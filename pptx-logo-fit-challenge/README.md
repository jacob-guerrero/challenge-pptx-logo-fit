# PPTX logo fit — technical challenge

Standalone exercise. You can zip this folder and run it anywhere with Node.js 18+.

## Problem statement

We brand PowerPoint templates by replacing a logo image inside the `.pptx` (ZIP + XML). The template has **one** picture placeholder on slide 1 with alt text:

```text
{d.unifiedLogo:imageFit(contain)}
```

The baseline script swaps the embedded PNG but **does not** adjust the picture frame. When the replacement logo has a different aspect ratio than the placeholder frame, the logo looks **stretched or squashed** in PowerPoint.

Your task: produce three output files where each logo looks correct (no deformation):

| Output file | Expected logo aspect |
|-------------|----------------------|
| `output/branded_square.pptx` | Square |
| `output/branded_h_rectangle.pptx` | Horizontal |
| `output/branded_v_rectangle.pptx` | Vertical |

### If you bring your own artwork

Put your three PNG files in **`assets/logos/`** with these exact names:

- `square.png`
- `h_rectangle.png`
- `v_rectangle.png`

Recommended pixel sizes (not required, but helpful):

- `square.png` — e.g. 400×400
- `h_rectangle.png` — e.g. 600×200
- `v_rectangle.png` — e.g. 200×600

### If you bring your own template

Replace **`assets/template/test_pptx_file.pptx`** only if you keep:

- A single slide with one `<p:pic>` whose alt/description contains `d.unifiedLogo`
- The same marker string used in `scripts/replace-logo.js` (`UNIFIED_LOGO_MARKER`)

## Quick start

```bash
cd pptx-logo-fit-challenge
npm install
npm run setup
npm run replace
```

Open the three files under `output/` in PowerPoint. You should see the deformation problem with the baseline.

### Run with one custom logo (optional)

Uses the same image for all three outputs (makes the frame mismatch obvious):

```bash
npm run replace -- --logo path/to/your-logo.png
```

### Run with reference logos (default)

Each variant uses its matching file under `assets/logos/`:

```bash
node scripts/replace-logo.js --use-reference-logos
```

## What we expect from you

1. Fix `scripts/replace-logo.js` and/or `assets/template/test_pptx_file.pptx`.
2. Document your approach in `SOLUTION.md` (create this file).
3. After `npm run replace`, all three outputs should show logos with correct proportions.

Hints (not the only paths):

- OOXML picture size lives in `<a:xfrm><a:ext cx="..." cy="..."/></a:xfrm>` inside the `<p:pic>`.
- “Contain” means fitting the image inside the frame while preserving aspect ratio (may letterbox).
- Production systems sometimes use **three separate placeholders** (one per `logoSize`) instead of one generic frame.
